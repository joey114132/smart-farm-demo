# 토마토 스마트팜 — 구성요소별 역할 (한국어)

> **참고:** [Robot8 Confluence — 페이지 5079065](https://robot8.atlassian.net/wiki/spaces/Robot8/pages/5079065) 는 로그인이 필요해 이 문서는 **실제 스타디움 모형 사진**, **100×200 cm 맵 분석도**, **`jetcobot_ros2/smart_farm` ROS 스택**, **브라우저 데모**를 맞춰 정리했습니다.

---

## 한눈에 보는 흐름

```
[북쪽·스타디움 밖] 토마토 재배대
        ↓ 수확
[스타디움 안] JetCobot A/B 작업셀 — 1차 적재(크레이트)
        ↓ 핸드오프
[스타디움 안] Pinky Pro — JetCobot 바로 아래 통로로 이동
        ↓ 남쪽 게이트
[스타디움 밖] 등급 판별 컨베이어 — 비전·A/B/폐기 분류
```

**핵심 원칙:** 재배·수확·단거리 이송은 **스타디움(100×200 cm 백색 벽 안)** 에서, **등급 판별·선별**은 **먼지·습기·혼류를 줄이기 위해 스타디움 밖(남쪽)** 컨베이어에서 처리합니다.

---

## 1. 스타디움 (STADIUM) — 100 cm × 200 cm

| 항목 | 설명 |
|------|------|
| **무엇** | 백색 벽으로 둘러싼 실내 작업장. 바닥에 라인 트랙·재배 모듈이 들어가는 **메인 필드** |
| **왜** | 로봇·작물·사람 동선을 한 좌표계(`map` 프레임)에 고정. Nav2·AMCL이 안정적으로 동작하는 범위 |
| **데모에서** | 흰색 테두리 사각형, 중앙에 `STADIUM (100×200 cm)` 라벨 |

**신뢰 로컬라이제이션 존 (분홍 띠)**  
벽에서 **15 cm 이상** 떨어진 구역. LiDAR·AMCL이 벽에 붙어 오차가 커지는 것을 피합니다. Pinky Pro는 가능한 이 안에서 주행합니다.

---

## 2. 북쪽 재배대 (스타디움 밖 · y > 200 cm)

| 항목 | 설명 |
|------|------|
| **무엇** | JetCobot 팔이 **담장 너머** 손이 닿는 토마토 재배 구역 (실제 모형의 상단 재배 트레이와 대응) |
| **누가** | **JetCobot A** → A열 재배대, **JetCobot B** → B열 재배대 |
| **왜 밖** | 팔 기지는 스타디움 **안** 작업셀에 두고, 작물만 **밖**에 두면 수확 시 스타디움 내부 혼잡을 줄일 수 있음 |
| **ROS** | MoveIt `ready_to_pick` 포즈, 비전으로 익은 열매 선택 (하드웨어: `mycobot_280_pi5_moveit_config`) |

---

## 3. JetCobot (고정형 수확·1차 적재)

| 항목 | 설명 |
|------|------|
| **하드웨어** | Elephant Robotics **MyCobot 280 Pi** + 적응형 그리퍼 (Pi 5에서 `pymycobot` 브리지) |
| **위치** | 작업셀 40×40 cm — **A (25, 170)**, **B (75, 170)** cm |
| **역할** | ① 북쪽 재배대 **수확** → ② 작업셀 남쪽 **크레이트에 1차 투입** (등급 최종 판정은 하지 않음) |
| **왜 2대** | 모형의 **Seeding/Planting**·**Harvesting/Sorting** 스테이션을 토마토 라인 **A/B** 로 나눈 것과 같음 |
| **주의** | A 셀 북서 모서리는 15 cm 클리어런스와 겹침 — 실제 배치 시 2~3 cm 안쪽 이동 권장 |

**SRDF 대표 포즈 (`jetcobot_ros2`)**

| 포즈 | 용도 |
|------|------|
| `home` | 대기·스토우 |
| `ready_to_pick` | 재배대 방향 전진 수확 |
| `ready_to_see` | 카메라·검사 |

---

## 4. Pinky Pro (이동형 운반)

| 항목 | 설명 |
|------|------|
| **하드웨어** | 차동 구동 모바일 베이스, RPLidar C1, 전방 카메라 (URDF: `pinkypro.urdf`) |
| **대수** | 데모 3대 — **P1 (20,30)**, **P2 (80,30)**, **P3 (50,80)** 앵커 기준 위상 차이 주행 |
| **역할** | JetCobot가 채운 **크레이트를 스타디움 남쪽 게이트까지 운반** → 컨베이어 투입구에 핸드오프 |
| **주행 규칙** | **시계 방향 단일 루프** — 정면 충돌·교착·Nav2 재계획 폭주 방지 |
| **JetCobot 아래 통로** | 북쪽 구간은 **y ≈ 138 cm** — 작업셀 남쪽 가장자리(y=150) **바로 아래**. 팔 스테이션(y=170) 안으로 들어가지 않음 |
| **ROS** | `vicpinky_navigation` / Nav2 `navigate_through_poses`, costmap inflation ≈ 18 cm |

**남쪽 외부 구간 (컨베이어 연결)**  
P2 쪽에서 **y = −10 cm** 게이트를 나와 컨베이어 **투입** 지점으로 진입한 뒤, 다시 스타디움 안으로 합류합니다.

---

## 5. 등급 판별 컨베이어 (스타디움 밖 · 남쪽)

| 항목 | 설명 |
|------|------|
| **위치** | 스타디움 **남쪽 벽(y=0) 밖**, y ≈ −30 cm 중심의 가로 벨트 |
| **왜 밖** | ① 수확 구역과 **선별·포장 공정 분리** ② 비전 조명·먼지 환경 분리 ③ Pinky·팔과 물리적으로 간섭 최소화 |
| **구성** | 투입 → 비전 등급 → A등급 / B등급 / 폐기 슈트 |

| 스테이션 | x (cm) | 하는 일 |
|----------|--------|---------|
| **투입** | 18, −10 | Pinky가 크레이트에서 토마토를 벨트에 올림 |
| **비전 등급** | 50, −30 | RGB/크기/결함 검사 — 등급·라우팅 결정 |
| **A등급** | 82, −38 | 상품용 출하 |
| **B등급** | 82, −30 | 가공용·할인 판매 |
| **폐기** | 82, −22 | 손상·미숙·이물 |

**하드웨어 연동 예시 (미구현·설계)**  
벨트 모터 + PLC/ROS2 `diagnostic_msgs` 또는 MQTT, 비전은 JetCobot/Pinky 카메라 또는 고정 산업 카메라, 분기는 서보 게이트.

---

## 6. 양액 혼합·저장 (스타디움 남쪽 벽 · 긴 탱크)

| 항목 | 설명 |
|------|------|
| **위치** | 스타디움 **남쪽 벽**을 따라 긴 직사각형 (대략 x 22~78, y 8~26 cm) — 모형의 노란·흰 스트라이프 존 |
| **무엇** | 양액 **믹서 + 저장 탱크** (고정 설비). 수경 재배에 쓰는 영양액을 섞어 두는 구역 |
| **역할** | 재배대·모듈로 **펌프·밸브**를 통해 공급 (Pinky가 운반하지 않음) |
| **Pinky와 관계** | 탱크 **위쪽** (y ≈ 32 cm)이 녹색 사각 주행 루프의 **남변**. 로봇은 탱크를 **지나가지 않음** |

---

## 7. 단계별 시나리오 (데모 타임라인)

| 단계 | 담당 | 설명 |
|------|------|------|
| 1. 대기 | JetCobot | `home`, Pinky 루프 순찰 |
| 2. 수확 | JetCobot | 북쪽 재배대에서 그리퍼 픽 |
| 3. 1차 투입 | JetCobot | 작업셀 크레이트에 적재 (등급 X) |
| 4. 이송 | Pinky Pro | JetCobot **아래 통로** → 남쪽 게이트 |
| 5. 컨베이어 투입 | Pinky + 벨트 | 스타디움 **밖** 투입구 |
| 6. 등급 판별 | 컨베이어 + 비전 | A / B / 폐기 분류 |

## 9. Confluence 문서에 넣기 좋은 한 줄 요약

> **스타디움 안**에서 JetCobot이 북쪽 재배대를 수확하고, Pinky Pro는 **양액 탱크 위** 녹색 사각 루프로 순찰·이송하며 JetCobot 작업셀 **바로 아래 통로**를 지난 뒤 **남쪽 밖 등급 컨베이어**에서 A/B/폐기를 판정한다.
